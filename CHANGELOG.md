# Changelog

## v0.2.0-alpha.1

- Remove non-user-facing prompt files from the distributable package.
- Add bilingual documentation via `README.zh-CN.md`.
- Add stable hash suffixes to diff-generated block/relation IDs to avoid collisions for non-ASCII or heavily sanitized paths.
- Add diff parser regression tests for non-ASCII paths, emoji paths, and multiple hunks per file.
- Clarify current renderer/export support: HTML render and Markdown export today; DOCX/PDF/JSON-LD remain roadmap items.


### Security hardening
- **P0 fixed:** `from-git` now uses `execFileSync` instead of `execSync` to prevent shell injection via `--base`/`--head` arguments.
- **P1 fixed:** `--atomic` + `--dry-run` now outputs the original unmodified document (not the partially-modified one) when operations are rejected.
- **P1 fixed:** Diff parser handles deleted files (`+++ /dev/null`) correctly.
- **P1 fixed:** Generated block IDs are now path-based (not index-based) for stability across re-generation.
- SECURITY.md updated with diff content injection and shell injection notes.


- Added `copctl from-diff <diff-file>` to generate a structural COP `code_review` object from a unified diff.
- Added `copctl from-git --base <ref> --head <ref>` to generate a COP review object from `git diff`.
- Added `src/diff.ts` with a minimal unified diff parser and COP document generator.
- Added `copctl context --with-hash` to include `preconditions.target_hash` in context packets and prompts.
- Added `copctl context --max-tokens` rough budget warning.
- Added `copctl context --include-relation-type` and `--exclude-relation-type` filters.
- Added `copctl apply-op --atomic` to reject a batch if any operation is skipped.
- Added `examples/e2e-code-review/generated-from-diff.cop.json` generated by the v0.2 diff workflow.
- This is an alpha workflow release; protocol document schema remains compatible with v0.1 COP documents.

## v0.1.3 â€” Security & Engine refactor

### Security fixes
- **P0 fixed:** `applyJsonPatch` now blocks prototype pollution via `__proto__`, `constructor`, `prototype` keys in JSON Pointer paths.
- **P1 fixed:** `object_merge` (`update_block`) now forbids overwriting `block.id`; warns if `block.type` is changed.
- **P1 fixed:** `add_relation` now validates that `from` and `to` references exist in the document before applying.
- **P1 fixed:** `create_block` warns when an agent creates an `instruction` block.
- **P2 fixed:** `resolve_comment` now validates `status` against the allowed enum before applying.
- **Security:** `cop context --prompt` now wraps block content in `<cop:block>` XML tags to reduce prompt injection surface.
- **Security:** `cop context` now excludes `instruction` blocks by default; use `--include-instructions` to override.

### New features
- `cop context --prompt-out <file>` â€” write prompt to a file instead of stderr, preventing stdout/stderr mixing.
- `cop context --include-instructions` â€” explicit opt-in to include instruction blocks in context packet.
- `cop hash --target <blockId>` â€” prints the stable SHA-256 hash of a block for use as `preconditions.target_hash`.
- **Replay protection:** `apply-op` now rejects operations whose `id` already exists in the document's `operations[]`.

### Engine refactor
- Operation logic extracted from CLI into `src/engine.ts` (`applyOperations`, `hashEntity`, `decodePointer`, `applyJsonPatch`, `mergePatch`). CLI is now IO-only.

### Protocol changes
- `trust_level: "verified"` removed from v0.1 schema (reserved for v0.2). Documents using `verified` will fail validation.
- `comments_on` removed from relation type enum (redundant with `comment.target`).
- `context_packets` top-level field removed from `cop.schema.json` (context packets are derived/transient, not stored in documents).
- `research-memo.cop.json` example: removed redundant `blk_rm_title` block (object.title already carries this).

### Docs
- `CHANGELOG.md` added.
- `ARCHITECTURE.md` updated: concurrent edit safety caveat, engine separation note.
- `SECURITY.md` updated: prototype pollution note, instruction block guidance.
