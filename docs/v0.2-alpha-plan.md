# COP v0.2 Alpha Plan

v0.2 alpha is intentionally not a broad protocol expansion. It focuses on the first concrete workflow loop:

```text
git diff → COP code_review object → context packet → model operation patch → apply-op → validate → render
```

## What is new

- `copctl from-diff <diff-file>`: create a structural COP object from unified diff text.
- `copctl from-git --base main --head HEAD`: create a COP object from `git diff`.
- `copctl context --with-hash`: include `preconditions.target_hash` in packets and prompts.
- `copctl context --max-tokens`: warn when rough context size exceeds a budget.
- `copctl context --include-relation-type` / `--exclude-relation-type`: filter context relations.
- `copctl apply-op --atomic`: fail the whole batch if any operation is skipped.

## Explicit non-goals

- `from-diff` does not perform semantic code review.
- v0.2 alpha does not call model APIs directly.
- v0.2 alpha does not provide cryptographic trust.
- v0.2 alpha does not guarantee multi-writer concurrency safety.
- v0.2 alpha does not provide an online validator yet.

## Review focus

Reviewers should focus on:

1. Whether `from-diff` creates useful enough COP objects.
2. Whether extension block types like `x-diff_hunk` are the right shape.
3. Whether context packets with `--with-hash` are ready for model workflows.
4. Whether `--atomic` is sufficient or misleading without document revision.
5. Whether v0.2 alpha overclaims usefulness before semantic review exists.
